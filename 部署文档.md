## 二、创建GitHub仓库

### 2.1 注册GitHub账号

1. 打开浏览器，访问：https://github.com
2. 点击 **"Sign up"**（注册）
3. 填写注册信息：
   - **Email**：您的邮箱地址
   - **Password**：密码（至少15位，或至少8位包含数字和字母）
   - **Username**：用户名（英文，将成为您仓库的地址一部分）
4. 验证邮箱（GitHub会发送验证邮件到您的邮箱）
5. 完成注册

### 2.2 创建新仓库

1. 登录GitHub后，点击右上角 **"+"** → **"New repository"**

   ![GitHub创建仓库](data:image/svg+xml;base64,PHN2Zy8L3N2Zy8+)

2. 在创建仓库页面，填写以下信息：

   | 字段 | 填写内容 |
   |------|----------|
   | Repository name | `task-management-system` |
   | Description | 任务管理系统 - 基于Vue3开发 |
   | Public/Private | 选择 **Public**（公开） |
   | Add a README file | **不要勾选** |
   | Add .gitignore | 选择 **None** |
   | Choose a license | 选择 **MIT License** |

3. 点击 **"Create repository"**（创建仓库）

4. **重要**：创建成功后，页面会显示仓库地址，复制备用
   
   仓库地址格式：`https://github.com/您的用户名/task-management-system.git`
   
   ![仓库地址](data:image/svg+xml;base64,PHN2Zy8L3N2Zy8+)

## 三、安装和配置Git

### 3.1 下载Git

1. 访问Git官网：https://git-scm.com
2. 点击 **"Download for Windows"**
3. 下载 `Git for Windows` 安装包

### 3.2 安装Git

1. 双击下载的安装包（文件名如 `Git-2.44.0-64-bit.exe`）
2. 安装向导步骤：

   | 步骤 | 操作 |
   |------|------|
   | 许可协议 | 点击 **"Next"** |
   | 安装位置 | 使用默认路径，点击 **"Next"** |
   | 组件选择 | 保持默认，点击 **"Next"** |
   | 开始菜单 | 点击 **"Next"** |
   | 编辑器选择 | 选择 **"Use Visual Studio Code as Git's default editor"**，点击 **"Next"** |
   | PATH环境 | 选择 **"Git from the command line and also from 3rd-party software"**，点击 **"Next"** |
   | SSH执行 | 选择 **"Use the OpenSSH server"**，点击 **"Next"** |
   | 行末换行符 | 选择 **"Checkout Windows-style, commit Unix-style line endings"**，点击 **"Next"** |
   | 终端模拟器 | 选择 **"Use MinTTY"**，点击 **"Next"** |
   | 默认行为 | 保持默认，点击 **"Next"** |
   | 凭据助手 | 选择 **"Git Credential Manager"**，点击 **"Next"** |
   | 实验功能 | 不勾选，点击 **"Install"** |
   | 安装完成 | 取消勾选 **"Launch Git Bash"**，点击 **"Finish"** |

3. 安装完成后，关闭安装窗口

### 3.3 验证Git安装

1. 按 `Win + R`，输入 `cmd`，回车
2. 在命令提示符中输入：
   ```bash
   git --version
   ```
3. 如果显示类似 `git version 2.44.0`，说明安装成功

## 四、上传代码到GitHub

### 4.1 打开项目目录

1. 按 `Win + R`，输入 `cmd`，回车
2. 进入项目目录：
   ```bash
   cd c:\课程作业或者论文\web
   ```
3. 验证目录是否正确：
   ```bash
   dir
   ```
   应该能看到 `src/`、`dist/`、`package.json` 等文件和文件夹

### 4.2 运行上传脚本

**双击文件**：`上传到GitHub.bat`

按照脚本提示操作：

```
============================================
     Git 上传脚本 - 任务管理系统
============================================

[1/6] ✓ Git 环境正常

请输入您的 GitHub 用户名: _
```

1. **输入GitHub用户名**：输入您在第二章注册的GitHub用户名
2. **输入邮箱**：输入您的邮箱地址
3. **粘贴仓库地址**：粘贴您在第二章创建的仓库地址
   
   ```
   https://github.com/您的用户名/task-management-system.git
   ```
4. **输入提交信息**：可以直接回车使用默认值
5. **登录GitHub**：如果提示登录，输入您的GitHub用户名和密码

### 4.3 常见上传问题

#### 问题1：提示"git commit"失败
```
*** Please tell me who you are.
```
**解决方法**：
```bash
git config user.name "您的用户名"
git config user.email "您的邮箱"
```

#### 问题2：提示权限错误
```
Permission denied (publickey).
```
**解决方法**：
- 使用HTTPS方式推送（输入用户名和密码时使用GitHub Token替代密码）
- GitHub已于2021年不再支持密码验证，需要使用Personal Access Token

#### 问题3：GitHub密码验证
- 密码栏输入：**GitHub Personal Access Token**
- 获取Token方法：
  1. 登录GitHub
  2. 点击头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)
  3. 点击 "Generate new token (classic)"
  4. 设置名称（如"task-management"），勾选 "repo" 权限
  5. 点击 "Generate token"
  6. **复制Token**（只显示一次，妥善保管）

## 五、部署到Vercel

### 5.1 注册Vercel账号

1. 打开浏览器，访问：https://vercel.com
2. 点击右上角 **"Sign Up"**
3. 选择 **"Continue with GitHub"**
4. 点击 **"Authorize Vercel"** 授权GitHub账号

### 5.2 导入GitHub仓库

1. 登录Vercel后，点击 **"Add New Project"**（添加新项目）

   ![Vercel添加项目](data:image/svg+xml;base64,PHN2Zy8L3N2Zy8+)

2. 在 "Import Git Repository" 页面，找到您的仓库：
   
   - 如果仓库在列表中：直接点击仓库名
   - 如果仓库不在列表中：点击 **"Import All"** 或搜索仓库名

3. 点击仓库名后，Vercel会自动检测项目配置

### 5.3 配置部署选项

在部署配置页面，保持默认设置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Framework Preset | Vue.js | 自动识别 |
| Build Command | `npm run build` | 默认 |
| Output Directory | `dist` | 默认 |
| Root Directory | `.` | 默认 |

点击 **"Deploy"** 开始部署

### 5.4 等待部署完成

部署过程会显示进度条，通常1-2分钟完成。

部署成功后会显示：
```
✅ Deployment Complete
🎉 Your site has been deployed!
```

页面会显示您的网站地址，如：
```
https://task-management-system.vercel.app
```

## 八、常见问题

### Q1：网站打不开，显示404

**原因**：Vercel还在部署中，或部署失败

**解决方法**：
1. 等待1-2分钟
2. 刷新页面
3. 检查Vercel项目中的部署记录

---

### Q2：修改代码后网站没有更新

**原因**：代码没有正确上传到GitHub

**解决方法**：
1. 确认GitHub仓库中的代码已更新
2. 检查Vercel的部署记录
3. 手动触发重新部署：在Vercel项目页面点击 **"Deployments"** → 选择最新部署 → 点击 **"Redeploy"**

---

### Q3：Vercel访问慢或打不开

**原因**：Vercel服务器在国外，国内访问可能较慢

**解决方法**：
1. 耐心等待加载
2. 尝试使用手机热点访问
3. 考虑使用国内替代平台（如腾讯云、阿里云）

---

### Q4：任务数据丢失

**原因**：数据保存在浏览器localStorage中

**说明**：
- 这是正常行为
- 每个用户的任务是独立保存的
- 清除浏览器缓存会删除任务数据

---

### Q5：如何删除网站

**在Vercel删除**：
1. 登录Vercel
2. 进入项目 → Settings → General
3. 点击 **"Delete Project"**
4. 输入项目名确认删除

**在GitHub删除**：
1. 登录GitHub
2. 进入仓库 → Settings
3. 拉到页面底部，点击 **"Delete this repository"**
4. 输入仓库名确认删除
